# Humanize Technical Text

기술 문서를 자연스럽게 만들 때는 문체만 만지고 정보는 건드리지 않는 것이 핵심입니다.

## Keep

- 용어
- 수치
- 논리
- 전제 조건

## Improve

- 첫 문단 진입 장벽
- 문장 리듬
- 접속어 과잉
- bullet 과잉
- 의미 없는 추상어

## Good Rewrite Test

다듬은 뒤 아래 질문에 예라고 답할 수 있어야 합니다.

- 더 쉽게 읽히는가
- 원래 뜻이 바뀌지 않았는가
- 사람이 실제로 설명하는 것처럼 들리는가
- 중요한 용어가 사라지지 않았는가
